> 本文档对应系统模块端口连接图（`system_module_port_connection.png`），覆盖从数据准备、功能仿真、结果验证到综合上板的全流程。

## 一、系统架构概览

本系统为 **FPGA 实现的 320×320 灰度 JPEG-like 压缩流水线**，采用纯 RTL 设计，数据流依次经过以下 7 个模块：

| 序号 | 模块名 | 功能 | 说明 |
|------|--------|------|------|
| 1 | **Y** | RGB565 → 亮度 Y | 将 16-bit RGB565 像素转换为 8-bit 灰度值 |
| 2 | **image** | 320×320 分块 | 将整帧图像按行优先输出，形成 8×8 像素块序列 |
| 3 | **block_fifo** | 像素缓冲 | 跨时钟域/速率匹配缓冲，向上游反压 (`ready_out`) |
| 4 | **dct_2d_controller** | 顶层状态机 | 控制 2D-DCT 的行/列运算流程，协调 FIFO 读写 |
| 5 | **dct_2d** | 8×8 二维 DCT | 采用 AAN 蝶形算法的定点 1D-DCT，行列各做一次 |
| 6 | **quantizer** | 标量量化 | 使用标准 JPEG 亮度量化表，输出 12-bit 有符号系数 |
| 7 | **zigzag_scan** | Zig-Zag 扫描 | 将 8×8 系数矩阵按 Zig-Zag 顺序展开为 64 个输出 |
| 8 | **jpeg_entropy_encoder** | DC/AC + Huffman | 对量化系数进行差分编码 + 变长编码 (VLC) |

> **注：`clk` 和 `rst_n` 连接至所有模块，图中未逐一画出。**

### 输出接口

系统有两组输出：
- **Zig-Zag 调试输出** (`out_en`, `out_data[11:0]`, `out_idx[5:0]`)：用于验证中间结果
- **熵编码输出** (`entropy_valid`, `entropy_code[15:0]`, `entropy_len[4:0]` 等)：最终压缩码流

## 五、功能仿真（ModelSim）

### 5.1 启动 ModelSim

打开 **ModelSim-Altera** 或 **QuestaSim**，确保工作目录为 `E:/fpga/jpeg_compress_gray`。

### 5.2 执行仿真脚本

在 ModelSim Transcript 窗口中执行：

```tcl
do sim/sim.do
```

**脚本自动完成以下操作：**
1. 创建/刷新 `rtl_work` 库
2. 编译全部 RTL 文件（含修复后的 DCT1D、量化器、熵编码器）
3. 编译 Testbench `tb_jpeg_compress.v`
4. 启动仿真，加载波形分组（TOP / DCT1D / DCT1D_butterfly）
5. 运行仿真直到 `frame_done` 或超时

### 5.3 观察波形

仿真脚本已预配置以下波形组：

- **TOP 组**：`clk`, `rst_n`, `en_in`, `rgb565_in`, `ready_out`, `out_en`, `out_data`, `out_idx`, `block_done`, `frame_done`
- **DCT1D 组**：输入输出、状态机、采样索引、蝶形输入 `x0~x6`
- **DCT1D_butterfly 组**：蝶形中间结果 `b0~b3`, `c0~c3` 等

### 5.4 仿真输出文件

仿真完成后，在 `sim/` 目录下生成：

| 文件 | 内容 | 格式 |
|------|------|------|
| `zz_out_all.txt` | 硬件 Zig-Zag 系数 | 每行：`idx data`（idx=0..63, data=有符号12bit） |
| `entropy_out.txt` | 硬件熵编码符号 | 每行：`is_dc is_eob code len extra extra_len` |

## 八、顶层模块端口详解

以下为 `jpeg_compress_top.v` 的完整端口说明，与连接图一一对应：

### 8.1 输入端口

| 信号名 | 位宽 | 方向 | 说明 |
|--------|------|------|------|
| `clk` | 1 | Input | 系统时钟（建议 100 MHz） |
| `rst_n` | 1 | Input | 低电平复位 |
| `en_in` | 1 | Input | 输入像素有效（高有效） |
| `rgb565_in` | 16 | Input | RGB565 像素数据 {R[15:11], G[10:5], B[4:0]} |

### 8.2 输出端口（控制与调试）

| 信号名 | 位宽 | 方向 | 说明 |
|--------|------|------|------|
| `ready_out` | 1 | Output | 可接收像素（高=可接收；低=反压上游暂停） |
| `out_en` | 1 | Output | Zig-Zag 系数有效 |
| `out_data` | 12 | Output | 量化后的 Zig-Zag 系数（有符号） |
| `out_idx` | 6 | Output | 系数在 8×8 块中的 Zig-Zag 索引 0..63 |
| `block_done` | 1 | Output | 一个 8×8 块处理完成（1 周期脉冲） |
| `frame_done` | 1 | Output | 整帧 1600 块处理完成（1 周期脉冲） |

### 8.3 输出端口（熵编码码流）

| 信号名 | 位宽 | 方向 | 说明 |
|--------|------|------|------|
| `entropy_valid` | 1 | Output | 熵编码符号有效 |
| `entropy_is_dc` | 1 | Output | 当前为 DC 符号（1=DC, 0=AC） |
| `entropy_is_eob` | 1 | Output | 当前为 EOB (End-of-Block) 标记 |
| `entropy_code` | 16 | Output | Huffman 编码前缀（VLC 码） |
| `entropy_len` | 5 | Output | `entropy_code` 的有效位长度 |
| `entropy_extra` | 12 | Output | 附加位（幅度值的原始位） |
| `entropy_extra_len` | 4 | Output | `entropy_extra` 的有效位长度 |

### 8.4 时序图（关键信号）

### 8.5 使用要点

1. **按 `ready_out` 反压送数据**：上游必须检查 `ready_out`，低电平时暂停发送像素
2. **每帧 102400 像素**：320×320 = 102400 个 RGB565 像素，必须送满整帧
3. **复位后等 2 周期再送数据**：`rst_n` 释放后，建议空 2 个时钟周期再拉 `en_in`
4. **熵编码输出无反压**：下游需实时接收或自行做 FIFO 缓冲
5. **量化表固定**：使用标准 JPEG 亮度量化表，如需调整请修改 `quantizer.v`

## 九、常见问题

### Q1: 仿真提示 `image_320x320_rgb565.hex` 找不到？

```bash
# 先准备数据
python sim/png_to_sim_input.py out/02.png out/02
```

### Q2: Zig-Zag 系数与参考模型不一致？

- 检查 `DCT1D.v` 舍入方式是否为 `(y+512)>>>10`
- 检查 `dct_2d.v` 列 DCT 输入位宽是否为 12-bit
- 检查 `quantizer.v` 最小值截断是否为 `MIN_VAL=-2047`

### Q3: 熵编码输出码流无法解码？

- 检查 `jpeg_entropy_encoder.v` 的 `category` 查表是否修复越界
- 确认 `huffman_tables.vh` 与标准 JPEG 表一致

### Q4: 综合时报 `multiply` 相关错误？

确保在 Quartus 中正确生成了 `arm` 乘法器 IP，且 `arm.qip` 已加入工程。
